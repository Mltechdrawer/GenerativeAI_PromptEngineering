    B 1: Key Concepts

    1. Artificial Intelligence (AI)

    Do you think ChatGPT thinks like a human?

    Artificial Intelligence (AI) refers to systems that perform tasks typically associated with human intelligence, such as understanding language, recognizing patterns, or making decisions.

    AI systems do not think or understand in a human sense. Instead, they learn patterns from large amounts of data and use those patterns to produce outputs.

    Think of AI as a student who has read millions of books and can answer questions based on that experience, but without true understanding.

    Examples - Recommendation systems (e.g., Netflix, Spotify) - Automatic translation tools
    Think about it Which AI systems do you use in your daily life without noticing?

    2. Machine Learning (ML)

    How can a machine learn without being explicitly programmed?

    Machine Learning (ML) is a subset of AI in which systems learn from data instead of following predefined rules.

    Instead of writing explicit instructions, we provide examples and allow the system to infer patterns and make predictions.

    This is similar to teaching a child what a dog is by showing many examples, rather than defining all its characteristics.

    Examples - Spam detection - Image classification
    Try it How would you teach a machine to distinguish between cats and dogs?

    3. Deep Learning

    What happens when problems become too complex for simple models?

    Deep Learning is a specialized area of Machine Learning that uses neural networks with multiple layers to process complex data.

    Each layer extracts increasingly abstract features, allowing the system to handle tasks that are difficult to solve with traditional approaches.

    You can think of it as a sequence of filters, where each layer refines the information further.

    Examples - Face recognition - Voice assistants (e.g., Siri, Alexa)
    Think about it Why is recognizing a face more complex than detecting spam?

    4. Generative AI

    What if AI could not only analyze information, but also create it?

    Generative AI refers to systems capable of producing new content such as text, images, or videos.

    These systems are trained on large datasets and can generate outputs that resemble human-created content.

    Think of it as an artist who has learned from thousands of works and can now create new ones.

    Examples - ChatGPT (text generation) - DALL·E (image generation)
    Try it Propose a creative prompt and see what kind of output an AI system can generate.

    5. Prompt

    How we ask matters.

    A prompt is the instruction or input given to an AI system to guide its response.

    The clarity and level of detail in a prompt strongly influence the quality of the output.

    Compare the following: - “Explain AI”
    - “Explain AI to a 12-year-old student using simple examples”

    Try it Rewrite a vague prompt to make it more specific and effective.

    6. Prompt Engineering

    What if asking the right question becomes a key skill?

    Prompt Engineering is the practice of designing and refining prompts to obtain better results from AI systems.

    It involves iterating on prompts by adding context, specifying formats, and introducing constraints.

    A simple prompt can be progressively improved: 1. Basic request
    2. Add context
    3. Specify output format
    4. Add constraints or conditions

    Try it Take a simple prompt and improve it step by step to obtain a more precise and useful response.
    Do language models only use patterns? What is the role of alignment?

    Large Language Models (LLMs) are fundamentally trained to learn patterns from vast amounts of text data. These patterns are not simple word associations, but complex relationships between concepts, structures, and contexts. In this sense, the model predicts the most likely continuation of a given input based on everything it has learned.

    However, this does not mean that the model “understands” in a human sense. It generates responses by leveraging statistical patterns that reflect how humans use language, reason, and explain ideas.

    This is where alignment becomes essential.

    Alignment does not teach the model new knowledge. Instead, it shapes how the model uses what it has already learned. Through techniques such as human feedback and instruction tuning, the model is guided to produce responses that are more useful, clear, safe, and aligned with user expectations.

    In simple terms:

    Pretraining teaches the model how to generate language.

    Alignment teaches the model how to behave.

    As a result, aligned models are better at following instructions, structuring answers, adapting tone, and avoiding harmful or irrelevant outputs. However, alignment does not eliminate the model’s limitations: it still relies on pattern prediction and may produce incorrect or misleading responses.

    In conclusion, language models rely on learned patterns, but alignment plays a crucial role in making those patterns usable, reliable, and appropriate for real-world applications.
