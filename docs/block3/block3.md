# Block III — Prompt Engineering

## Objective
Master techniques to control the **level of detail, format, and quality** of responses.

## Recommended structure of a “professional” prompt
1. **Context**
2. **Role**
3. **Task**
4. **Constraints**
5. **Output format**
6. **Example** (optional but very powerful)

### Reusable template

```text
Context:
Role:
Task:
Constraints:
Output format:
Quality criteria:
```

## Techniques we will practice
- Zero-shot vs few-shot.
- Step-by-step prompts (plan → execution → verification).
- Strong constraints (length, tone, tables, mandatory fields).
- Modular prompts (copyable blocks).
- Rapid iteration (refinement with A/B testing).

## Guided practice (step by step)
Case: generate an **assessable activity** (or a report) with controlled output.

1. Base prompt (simple).
2. Add role + context.
3. Add constraints + format.
4. Add quality criteria + verification.
5. Save it as a reusable template.

## Common mistakes
- Ambiguous objective.
- Output format not specified.
- Lack of data or context.
- Not requesting verification / explicit assumptions.
